Project: Pydoit a task tracking application (python/gtk)

This desktop application has 2 parts - The end user who needs to be kept on track with the lists, and the list maker, which there is no user interface for, just a structured folder that contains plain text lists with associated icons and sounds.

The project code is here: https://github.com/semisided1/pydoit

Interesting things I could write about from this project:

  • Using glade to design a gui
  • Involving your children in application development
  • Python / sqlite / gtk

The pydoit user interface.

Title bar displays the current user name (this is a multi user application) and current list. The list on the left displays the completed tasks on the currently selected day. Clicking on a task in the list will undo the task. The calendar control displays which day is currently selected but also, if you click on a task the days that that task was completed will cause the dates in the calendar to appear bold. If a task hasn't been done, clicking on a button in the button panel will add it to the completed list and play the sound file assosiated with that task, as well display any instructions / information at the bottom of the window.

The pydoit configuration

As has been mentioned there is no user interface to create the lists. Pydoit expects the lists to follow a certain structure. Each list has its own folder. In that folder there is a text file of the same name which lists the tasks in the order that they should appear in the button panel. Benieth that there are folders each with the same name of each task. Within that folder there are 3 files - A sound file that gets played when the button is pressed. An image that will appear on the button. And some text that describes the task.
