Project: Photo Gallery on Appengine (java)

This web application has 2 parts - the website and the administration of the website. The website has a front page slide show, menu and gallery pages are generated from the admin user interface.

This is a little note to thank my patient customer - Steven - who did contribute a lot to the visual design of the site and of course the awesome photos.

The project code is here: https://github.com/semisided1/slpdotca

A live version of the web application is here: http://stevenlaytonsphotography.ca

Interesting things I could write about from this project:

  • Using the xmp tag and jquery to hide/unhide html fragments within a page
  • Using Saxonica s9api / xslt within Appengine
  • Using google feeds api
  • adding a facebook widget

This project was started when appengine was just released, before any python or node.js was available from google cloud. At this point java and xslt where still thought to be practical for small or medium sized websites. If you look at the code for this project you will be surprised at the complexity. A lot of that has to do with xml, google feeds and the hassle of passing around information into xslt. xslt is a great template engine and I generally prefer it to django / php style template systems. There are some real caveats to using xslt!!! Its not easy handling entity references, special characters (ie <!doctype .. ) and just making small changes can get your eyes seeing double: don't forget all your html has to be valid xml.

Overall feelings about this project:

the website has been in use for several years so despite the complexity it does achieve its main goal: the customer can change the photos without my help and is free of clutter from the website visitors perspective.
